# Linear

## Titles

**2–4 words.** Fewest words possible.

- **Good:** "User Facing Changelog", "QR Auto-Verify", "Bulk Upload Pipeline"
- **Bad:** "Add a changelog component that fetches GitHub releases"

## Context

Before creating issue, gather:

1. **List projects** — find correct project
2. **List existing issues** — identify relationships (deps, parent/child)
    > NEVER add as sub-issue to similar parent or "done" parent. Match what user currently working on
3. **List statuses and labels** for target team

Always populate: **status**, **priority**, **project**, **labels**, **parent issue** (if applicable).

No orphaned issues — every issue belong to project and hierarchy.

## Status Lifecycle

Update status proactively:

- No real plan → **Backlog**
- Planned → **Todo**
- Start work → **In Progress**
- Code complete → **In Review**
- **NEVER** set "Done". User job.
