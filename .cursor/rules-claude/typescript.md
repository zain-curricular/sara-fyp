# TypeScript

## Formatting

### Vertical Rhythm

- ONE blank line between main code chunks (headers, functions, constants). Split logically
- ONE blank line after opening brace `{` and between internal logical blocks

## Documentation

1. ALWAYS include file header documentation (below)
2. ALWAYS use JSDoc `/**...*/` for all exported symbols
3. ALWAYS use `//..` comments at top of small logical blocks to explain flow

### File Header

Every file get documentation block at top. Explain logic, integrations, architecture — reader understand what file do + why at glance.

```typescript
// ============================================================================
// Example File Name
// ============================================================================
//
// Overview of file here
//
// Example Info
// ------------
// Description of subsection here...


import c from eheu
...
```
